Контейнеры и блочные элементы

Блочные элементы — это строительные блоки HTML-страниц. Они занимают всю доступную ширину и начинаются с новой строки. В этой главе вы изучите универсальный контейнер <div>, семантические блоки <section> и <article>, а также специализированные блочные элементы: цитаты, предварительно форматированный текст, контактные данные и раскрывающиеся блоки.

Урок 1 — Элемент <div>

Тег <div> (от division — раздел) — это универсальный блочный контейнер без семантического значения. Он используется для группировки элементов с целью применения стилей или скриптов. <div> занимает всю доступную ширину и создаёт новую строку до и после себя.

💡
Когда использовать <div>? Используйте <div>, когда нет подходящего семантического элемента. Если содержимое имеет смысловое значение — предпочтите семантические теги (<section>, <article>, <nav> и т.д.).

Урок 2 — <section> и <article>

<section> и <article> — семантические блочные элементы HTML5. <section> обозначает тематически объединённый раздел страницы. <article> представляет самостоятельный, независимый контент (статью, пост, комментарий), который можно вырвать из контекста страницы.

ЭлементКогда использоватьПример
<section> Раздел страницы с общей темой «О нас», «Услуги», «Контакты»
<article> Самостоятельный контент Статья, пост в блоге, отзыв
<div> Нет семантики Обёртка для стилей/скриптов

Урок 3 — <blockquote>

Тег <blockquote> используется для длинных цитат из внешнего источника. Браузер отображает его с отступами. Атрибут cite указывает URL источника цитаты (не отображается визуально, но полезен для поисковиков). Для указания автора используют тег <cite> внутри <footer>.

citeURL источника цитаты (необязательный)

Урок 4 — <pre> и <code>

<pre> (preformatted) сохраняет все пробелы, переносы строк и форматирование как есть — браузер не сворачивает пробелы. Обычно используется вместе с <code> для отображения блоков программного кода. Одиночный <code> (строчный) показывает короткие фрагменты кода внутри текста.

💡
В тексте внутри <pre> нужно экранировать < как &lt; и > как &gt;, иначе браузер распознает их как HTML-теги.

Урок 5 — <address>

Тег <address> используется для контактной информации автора или организации. Содержимое обычно отображается курсивом. Элемент может содержать физический адрес, email, телефон, ссылки на профили в социальных сетях — всё, что связано с контактными данными. Используйте его в <footer> статьи или страницы.

⚠️
Важно: Тег <address> не предназначен для произвольных почтовых адресов в тексте — только для контактных данных владельца/автора текущего документа.

Урок 6 — <details> и <summary>

Тег <details> создаёт раскрывающийся виджет без JavaScript. Внутри него <summary> задаёт видимый заголовок/кнопку. По умолчанию содержимое скрыто; атрибут open делает его видимым сразу при загрузке страницы. Отлично подходит для FAQ, спойлеров, дополнительных деталей.
