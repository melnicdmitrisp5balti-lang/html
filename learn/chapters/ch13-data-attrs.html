Атрибуты и Data-атрибуты

Атрибуты добавляют элементам дополнительные сведения и поведение. Одни атрибуты специфичны для конкретных тегов (href у ссылки, src у изображения), другие — глобальные и работают на любом элементе. В этой главе вы изучите глобальные атрибуты, узнаете, как хранить произвольные данные в data-* атрибутах, познакомитесь с основами доступности через ARIA-атрибуты и научитесь работать с data-атрибутами через JavaScript.

Урок 1 — Глобальные атрибуты

Глобальные атрибуты можно применить к любому HTML-элементу. Они являются базой для стилизации, скриптинга, доступности и интернационализации.

АтрибутОписаниеПример
id Уникальный идентификатор на странице id="header"
class Класс (один или несколько, через пробел) class="btn btn-primary"
style Встроенные CSS-стили style="color: red;"
title Всплывающая подсказка при наведении title="Нажмите для подтверждения"
lang Язык содержимого элемента lang="en"
dir Направление текста: ltr, rtl, auto dir="rtl"
hidden Скрывает элемент (логический атрибут) hidden
tabindex Порядок перехода по Tab; -1 исключает из табуляции tabindex="0"
contenteditable Разрешает редактирование содержимого contenteditable="true"
draggable Делает элемент перетаскиваемым draggable="true"
spellcheck Проверка орфографии в редактируемых элементах spellcheck="false"
translate Разрешить/запретить перевод браузером translate="no"

id должен быть уникальным! На одной странице не может быть двух элементов с одинаковым id. Нарушение этого правила приводит к ошибкам в JavaScript и CSS, а также к проблемам с доступностью.

Урок 2 — Data-атрибуты (data-*)

Data-атрибуты позволяют хранить произвольные данные прямо в HTML-элементах. Они начинаются с data-, за которым следует любое имя: data-user-id, data-price, data-color. Данные невидимы пользователю, не влияют на отображение, но доступны JavaScript через dataset.

💡
Правило именования: имя после data- должно состоять только из строчных букв, цифр и дефисов. В JavaScript дефисы преобразуются в camelCase: data-user-id → dataset.userId.

Урок 3 — ARIA-атрибуты и доступность

ARIA (Accessible Rich Internet Applications) — набор атрибутов, улучшающих доступность для людей с ограниченными возможностями. Они сообщают скринридерам (программам для озвучивания экрана) о роли, состоянии и свойствах элементов. Семантические HTML-теги уже имеют встроенную доступность, но ARIA помогает, когда семантики недостаточно.

АтрибутНазначение
aria-label Текстовая метка элемента (для скринридера, когда нет видимого текста)
aria-labelledby Ссылка на id элемента, который служит меткой
aria-describedby Ссылка на id элемента с подробным описанием
aria-hidden="true" Скрыть элемент от скринридера (декоративные иконки)
aria-expanded Состояние раскрывающегося элемента (true/false)
aria-checked Состояние чекбокса/переключателя
aria-disabled Элемент недоступен для взаимодействия
aria-live Динамически обновляемая область (polite, assertive)
role Явная роль элемента: button, navigation, dialog, alert...

Урок 4 — Data-атрибуты и JavaScript

JavaScript читает и записывает data-атрибуты через свойство dataset. Это удобный способ передавать данные из HTML в скрипты без необходимости делать AJAX-запросы или хранить данные в глобальных переменных. Data-атрибуты также работают как хуки для CSS-стилей через CSS-селекторы атрибутов.

💡
dataset и camelCase:
data-user-id → element.dataset.userId
data-max-value → element.dataset.maxValue
data-price → element.dataset.price

Разметьте карточки товаров с data-атрибутами

Задание: Создайте HTML-страницу с тремя карточками товаров, используя атрибуты и принципы доступности:
  • Каждая карточка (<div> или <article>) должна иметь уникальный id
  • Каждая карточка должна иметь атрибуты data-price и data-category
  • Кнопка «Купить» в каждой карточке должна иметь aria-label с названием товара
  • Одна из карточек должна быть скрыта атрибутом hidden
  • Добавьте title хотя бы одному элементу с подсказкой
