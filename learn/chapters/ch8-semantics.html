Семантический HTML

Семантические теги HTML5 описывают смысл содержимого, а не только его внешний вид. Они помогают поисковикам лучше понимать структуру страницы, улучшают доступность для людей с ограниченными возможностями и делают код более читаемым.

💡
Семантика vs div: Раньше весь макет строился на тегах <div>. HTML5 ввёл смысловые теги: <header>, <nav>, <main>, <footer> и другие. Они несут в себе смысл — поисковики и читалки их "понимают".

Урок 1 — <header> и <footer>

<header> — шапка страницы или секции (обычно содержит логотип, заголовок, навигацию). <footer> — подвал страницы или секции (копирайт, ссылки, контакты). Оба тега можно использовать несколько раз на странице (например, в каждой статье).

Урок 2 — <nav> — навигация

Тег <nav> обозначает блок навигации — набор ссылок для перемещения по сайту. Не каждый список ссылок должен быть в <nav> — только основные навигационные блоки (главное меню, хлебные крошки, оглавление).

Урок 3 — <main>, <article>, <section>

<main> — основной уникальный контент страницы (только один на странице). <article> — самостоятельная единица контента (статья, пост, комментарий). <section> — тематический раздел страницы (обычно с заголовком).

Урок 4 — <aside> — боковая панель

Тег <aside> содержит вспомогательный контент, косвенно связанный с основным содержимым: боковые панели, блоки "Читайте также", реклама, цитаты, навигация по разделам.

Урок 5 — <figure> и <figcaption>

Тег <figure> обозначает самостоятельный блок контента (изображение, схема, код), на который можно ссылаться из основного текста. <figcaption> — подпись к этому блоку.

Урок 6 — <time> и <address>

<time> — машиночитаемая дата/время. Атрибут datetime содержит дату в стандартном формате (ISO 8601), а содержимое тега — для пользователя. <address> — контактная информация автора или организации.

Создайте страницу с семантической структурой

Задание: Создайте страницу блога с правильной семантической разметкой:

  • Тег <main> с основным контентом
