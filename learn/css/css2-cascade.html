Каскад и специфичность

Слово «каскадные» в названии CSS (Cascading Style Sheets) означает определённый алгоритм, по которому браузер решает, какой именно стиль применить к элементу, если несколько правил конфликтуют между собой. Понимание каскада и специфичности — ключевое условие предсказуемого и управляемого CSS.

Урок 1 — Что такое каскад

Каскад — это алгоритм, который браузер применяет при конфликте CSS-правил. Когда несколько правил подходят одному элементу, браузер определяет, какое из них «победит». Для правил с одинаковой специфичностью действует простое правило: последнее написанное правило побеждает.

💡
Каскад учитывает три фактора: источник стилей (браузер / автор / пользователь), специфичность селектора и порядок объявления. Каждый следующий фактор применяется только при равенстве предыдущего.

Урок 2 — Специфичность

Специфичность — это «вес» CSS-селектора. Браузер считает её как четырёхзначное число и использует для выбора «победившего» правила при конфликте. Чем выше специфичность, тем приоритетнее стиль.

Инлайн-стиль (style="")1-0-0-0<p style="color:red">
ID0-1-0-0#header
Класс / псевдокласс / атрибут0-0-1-0.nav, :hover, [type]
Тег / псевдоэлемент0-0-0-1p, ::before
Универсальный *0-0-0-0*

Урок 3 — Порядок правил и !important

При равной специфичности побеждает правило, объявленное позже в коде. Ключевое слово !important повышает приоритет объявления выше любой специфичности — но использовать его следует крайне редко.

⚠️
Осторожно с !important: Злоупотребление !important делает код непредсказуемым и трудно поддерживаемым. Используйте его только как крайнюю меру — например, для переопределения стилей сторонних библиотек.

Урок 4 — Источники стилей

CSS-стили могут поступать из трёх источников, и у каждого своя роль в каскаде. Приоритет от низшего к высшему: стили браузера → стили автора → стили пользователя (с учётом !important — порядок обратный).

💡
Стили браузера (User Agent Stylesheet) — встроенные стили по умолчанию: например, синие ссылки, отступы у заголовков, жирный шрифт у <b>. CSS-сброс (reset) или нормализация (normalize.css) устраняют эти различия между браузерами.
✅
Стили автора — это ваш CSS-файл или тег <style>. Они имеют наивысший приоритет из трёх обычных источников и перекрывают стили браузера.
